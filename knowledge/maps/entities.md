# Entities

Lookup notes for the named nouns of this domain — individual **plugins** (dev-workflow, meta-claude,
pm-workflow, knowledge-commons) and **skills**. Retrieved by name, never by association; each may
accumulate curated context and a dated interaction log.

## Plugins

- [[knowledge-commons plugin]] — generator plugin that scaffolds project-owned graph tooling; ships no verification machinery.

## Skills

- [[dev-workflow spec-writing skill]] — interview-driven spec skill; ran in main context to converge the knowledge-commons spec.
- [[graph-init skill]] — scaffolds the graph and generates the project's process + knowledge-graph skills; has a `--config-only` wiring mode.
- [[promote skill]] — uniform cross-graph promotion; `domain:`-stamps derived claims and commits-and-pushes them.
- [[the generated knowledge-graph skill]] — project-owned; defines note shapes, conventions, and the one structural rule (evidence supports an attractor).
- [[the generated process skill]] — project-owned orchestrator: resolve → ledger → inspect → plan → run → stamp → report, with an optional promotion tail.

## Concepts

- [[the commons]] — the single cross-domain graph at `~/Developer/commons`; receive-only via `/promote`, the evidence tier beneath the rules.
