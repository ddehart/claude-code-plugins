The finalized spec was audited with a single no-context implementer asked to plan the build — "can you do
this without clarifying questions?" — and explicitly not put through a second round. An adversarial,
repeated audit was precisely what had inflated the prior spec (363→990 lines while producing zero code):
an agent briefed to enumerate everything wrong returns a long list from any artifact, so the list measures
the brief, not the spec. The neutral build-it verdict is the one that measures whether the spec is
sufficient.
