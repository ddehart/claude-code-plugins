# processing a live session is not delegable to a subagent

Two independent reasons, either sufficient.

The pipeline's plan gate uses an interactive question tool unavailable in subagent contexts. A delegated run
would either stall at the gate or proceed without it — and proceeding without it is precisely what the gate
exists to prevent.

And a chronicle's reflection must be first-person. A subagent reading a transcript writes *about* someone
else's session: fluent, plausible, and not the thing the practice is for. **Testimony from someone who
wasn't there isn't testimony.**

The parallelism is still available, one level down: the inspection step fans out one reader per signal
class, which is where the volume actually is. What stays in the main context is the approval and the
reflection — the two things that depend on having been present.

*Decided 2026-07-22.*
