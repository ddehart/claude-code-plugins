# promotion is scheduled into the process pipeline, never left to memory

Any graph with a promotion target gets one addition to its generated process template: after the
extraction plan is assembled, the run asks *"does any of this generalize?"* and puts the candidates into
the same plan the human already approves. The design must never depend on anyone *remembering* to
promote — that is the failure mode that starves these systems. Scheduling the question into the pipeline
means it fires whenever the pipeline fires, on material that is already refined and in context, at
effectively zero extra cost.
